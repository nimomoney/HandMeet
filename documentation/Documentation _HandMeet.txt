 Documentation Technique : Système de Chat HandMeetVersion : 
1.0Statut : Optimisé (Vitesse & Budget)
1. Vue d'ensembleLe système de chat est conçu pour être rapide, temps réel et extrêmement économe en ressources Firestore. Il repose sur deux piliers : le temps réel pour les nouveaux messages et la pagination par cache pour l'historique.
2. Fonctionnalités Implémentées Optimisation du Budget (Lectures Firestore)Persistance Locale : Les données sont stockées dans le navigateur (IndexedDB). Un message déjà chargé une fois n'est plus jamais facturé par Google lors des prochaines consultations.Priorité au Cache : Lors du scroll vers le haut, l'application force la lecture dans la mémoire locale avant de solliciter le serveur.Vérification technique : Des logs en console confirment la source de données (CACHE ou SERVEUR). Gestion du Temps RéelSnapshot Intelligent : L'application écoute en permanence les 25 derniers messages. Dès qu'un message arrive, il s'affiche instantanément.Auto-Scroll : Le chat descend automatiquement lors de la réception d'un nouveau message, mais préserve la position de lecture lors du chargement de l'historique. Nettoyage Automatique (Sécurité & Coût)Système TTL (Time To Live) : Chaque message contient un champ expireAt réglé à 24 heures après sa création.Auto-suppression : Google Cloud nettoie automatiquement la base de données, évitant ainsi de stocker des milliers de messages inutiles sur le long terme.3.
